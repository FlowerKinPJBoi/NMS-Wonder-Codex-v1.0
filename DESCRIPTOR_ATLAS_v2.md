# Wonder Codex Descriptor Atlas v2

The Descriptor Atlas connects appearance tokens from an exact pet/discovery pair to controlled Wonder Projector evidence. It is designed to improve representative catalog art without ever presenting a reconstruction as the exact specimen.

## What this release does

- Reads descriptor tokens already retained in an approved exact pet match.
- Creates a stable `WCV-` visual-profile fingerprint from family plus sorted descriptor tokens.
- Exposes cautious token-name hints such as head, horns, tail, body, limbs, surface, or accessory.
- Shows coverage on public records while keeping VP hex values off the public record display.
- Ships an empty, evidence-gated Atlas registry and schema for future controlled projector correlations.

## What this release does not do

- It does not copy third-party editor mappings, lookup tables, source code, or imagery.
- It does not assert that a readable token name proves a body-part mapping.
- It does not call representative art an exact image.
- It does not treat appearance seeds as locations.

Only mappings supported by Wonder Codex-controlled projector captures should be added to `research/descriptor_atlas_v2.json`.
