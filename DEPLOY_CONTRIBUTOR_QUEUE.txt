WONDER CODEX CONTRIBUTOR QUEUE v0.3
===================================

This release removes the visible contributor submission key.

The contributor enters their name once at the top of the importer. After
analysis, the page displays that contributor name and offers:

    Submit to Wonder Database

IMPORTANT
---------
Submissions do NOT publish directly into the official Wonder Database.
They enter a PostgreSQL review queue with status "pending".

This gives contributors an easy experience while protecting the research data.

DEPLOYMENT
----------
1. Copy this package into the GitHub repository connected to DigitalOcean.
2. Push/commit the changes.
3. In the DigitalOcean App, add a SERVICE from the same repository.
4. Set Source Directory to:

       /api

5. Run command:

       uvicorn app:app --host 0.0.0.0 --port $PORT

6. HTTP port:

       8080

7. Route:

       /api

8. Add these API service environment variables:

   DATABASE_URL
       The PostgreSQL connection string from DigitalOcean.

   ALLOWED_ORIGINS
       https://wondercodex.com,https://www.wondercodex.com

   MAX_REQUESTS_PER_HOUR
       5

   IP_HASH_SALT
       Any long private random phrase. This is never shown to contributors.

9. Deploy and test:

       https://wondercodex.com/api/health

Expected mode:

       "review-queue"

10. Analyze a save at:

       https://wondercodex.com/import.html

The backend creates:

- submission_batches
- submitted_discoveries
- submitted_pet_matches
- submission_issues

PROTECTIONS INCLUDED
--------------------
- Raw character JSON stays in the browser.
- Only normalized records are submitted.
- Every submission is pending review.
- Stable hashes skip duplicate discoveries and pet matches.
- Five submissions per IP per hour by default.
- A hidden honeypot rejects simple form bots.
- PostgreSQL credentials stay in DigitalOcean environment variables.

This alpha is suitable for community contributions, but the next important
feature is an administrator review page for approving or rejecting batches.
