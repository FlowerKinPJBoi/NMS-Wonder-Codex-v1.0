WHAT WONDER CODEX READS

Clear data boundaries before you begin.

Read locally from the JSON

  • Save identity: save name and platform label.
  • DiscoveryData: discovery type, Universal Address, VP values, discovery owner, and platform.
  • PetData: Creature ID/type, Universal Address, creature/species/genus seeds, and secondary seed for exact DiscoveryData matching.
  • GenerationID records: counted for the analysis summary only.

Retained only after you press Submit

  • Submission details: contributor name, public/private attribution choice, save name, platform, and summary counts.
  • Normalized discoveries: DiscoveryData values, calculated Wonder Projector Message ID, owner/platform, and JSON record path.
  • Exact pet matches: Creature ID/type, matched discovery values, secondary-seed comparison, Message ID, and JSON record paths.
  • Review issues: matching or normalization notes needed by Wonder Codex reviewers.

Not retained: the original JSON, inventory, bases, missions, account identifiers, unrelated player progress, complete PetData or GenerationID records, or local file-system paths. A salted IP hash and browser user-agent are stored separately for rate limiting and security.
